## 〇、严禁直接复制未经润色的机翻！

机器翻译仅可作为辅助参考，**不得直接复制未润色的机翻结果**作为最终译文。所有译文必须经过人工润色和校对。

## 一、标点符号及数字

### 1.1 基本标点规则

| 规则 | 说明 |
|------|------|
| 中文标点 | 译文中使用中文全角标点 `（！？，。）`，而非英文半角 `(!?,.)` |
| 省略号 | 英文 `(...)` 翻译为全角省略号 `……`（Shift+^），而非三个句号 `。。。` |
| 破折号/省略号完整性 | 英文省略号和 `(-)` 翻译为中文 `……` 和 `——` 时，**请勿只写一半** `(—)(…)`，需确保完整 |

### 1.2 `(-)` 的处理

语句中的 `(-)` 根据语境翻译为：
- 逗号 `，`
- 省略号 `……`
- 破折号 `——`
- 表达口吃时可考虑不加标点

**禁止保留 `(-)` 原文。**

### 1.3 引号规则

| 规则 | 说明 |
|------|------|
| 单引号转换 | 英文成对单引号 `('xx')` 翻译为全角双引号 `"xx"`，**禁止**翻译为半角双引号 `("xx")`，视情况也可不加引号 |
| 独立引文（单段） | 段首和段尾都用引号 |
| 独立引文（多段） | 每段开头用前引号，只在最后一段末尾用后引号（GB/T 15834-2011 标点符号用法 4.8.3.5） |

### 1.4 换行符

- 译文的换行符应尽量与原文**完全保持一致**
- res 中缺换行问题不大
- src 中一般没有换行符 `\n`，需注意**保存快捷键按错**时出现**多余换行符**的情况

### 1.5 机翻代码格式修正

机翻辅助翻译时，需注意以下代码格式问题：

| 常见错误 | 修正为 |
|----------|--------|
| `.speak` / `.speaking` | `.speech` |
| 全角括号 `）` | 半角 `)` |
| 全角大方括号 `］` | 半角 `]` |

## 二、翻译技巧及建议

### 2.1 初次翻译

如果是**第一次翻译**，最好在翻译第一个词条后请有经验的编辑者检查，以便对常见问题及时处理。

### 2.2 键值提示

键值通常说明了各词条的作用，翻译时可作为参考依据。

### 2.3 长难句处理

- 长难句善用**拆句、调换语序**的方法翻译（例如各种从句）
- 具有逻辑关系的短句可合并
- 根据情况适当使用**删译**，在保留原文重要信息的情况下使译文顺畅

### 2.4 语言风格

- **禁止**使用方言词汇和流行语
- 同样的词不要在相近段落内密集使用
- 翻译对话的起始句时，按汉语习惯把**对人的称呼移至句首**

### 2.5 代词精简

- `my`、`your`、`a` 这类代词在不影响句子意思的情况下**能省则省**
- 物品数量和所有关系在上下文中显而易见时，严格照翻反而冗余
- `们` 在不影响句子意思的情况下能省则省，没必要见到复数就加
- `了` 同理，能省则省

### 2.6 "您"的使用

`您` 仅在说话者**必然**对倾听者使用尊重语气时使用。

### 2.7 术语管理

- 勤使用搜索原文和术语功能
- 有的术语存在多种翻译情况，请勤添加术语
- 可视讨论参考添加术语的情况

### 2.8 编辑建议

- 需要**大量编辑**长文本前，把词条标为**未翻译**，避免词条卡死
- 太长的文本建议在**本地编辑**，网页存在闪退可能

## 三、代码与符号处理策略

### 3.1 保留原样

| 符号 | 处理方式 |
|------|----------|
| `[]` 及其中字符代码 | **保留** |
| `{}` 及其中字符代码 | **保留** |
| `•` 符号及后面空格 | **保留**（选项符号） |

### 3.2 `()` 的处理

- 采用**英文半角符号**，主要防止超框现象
- `()` 内的内容正常翻译

### 3.3 其余符号

- 其余符号自行转换为**中文标点符号**，包括双引号 `""`、单引号 `''`
- 对话键值请参照上下文

## 四、翻译流程

### 4.1 标准流程

1. **选题评估** — 确认文档为第三方内容，评估翻译价值
2. **预处理** — 提取原文文本，确认文档结构
3. **术语确认** — 对照术语表，新增术语先登记
4. **初译** — 按章节翻译，保持一致性
5. **校对** — 逐句对照原文，核对规则数值
6. **润色** — 优化中文表达，统一风格
7. **终审** — 通读检查格式与排版

### 4.2 质量检查清单

- [ ] 标点符号使用正确（全角/半角）
- [ ] 省略号、破折号完整无缺
- [ ] 引号规则应用正确
- [ ] 换行符与原文一致
- [ ] 机翻代码格式已修正（`.speech`、半角括号等）
- [ ] 无方言词汇和流行语
- [ ] 术语翻译与术语表一致
- [ ] 游戏数值准确无误
- [ ] 代词精简到位
- [ ] 对话称呼位置正确

## 附录：常见问题

**Q: 省略号怎么打？**
A: 中文输入法下 `Shift+^`（即 `Shift+6`）打出全角省略号 `……`。

**Q: 破折号怎么打？**
A: 中文输入法下 `Shift+-` 打出全角破折号 `——`。注意是两个字符连在一起，不能只打一个 `—`。

**Q: 什么时候保留英文括号？**
A: 代码中 `()` 保留英文半角；中文正文中 `()` 也建议保留半角防止超框，内部正常翻译。

**Q: `.speak` 为什么会被改成 `.speech`？**
A: 机翻引擎有时会错误处理代码中的字符串，`.speak` 和 `.speaking` 在游戏代码中实际应为 `.speech`，翻译时需手动修正。
